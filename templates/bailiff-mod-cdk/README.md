# bailiff-mod-cdk

The **AWS CDK overlay** — scaffolds a CDK app via `cdk init` as a trust-gated
task. Pure TASK module: `template/` renders only the answers-file; ALL CDK
files come from `cdk init` (task-output). Supports TypeScript, Python, Go,
Java, and C# targets. Runs optionally after
[`bailiff-mod-base`](https://github.com/bailiff-io/bailiff-mod-base) (works
standalone). Spec 011 T021.

## What it produces

| Output | Lifecycle | Notes |
|---|---|---|
| CDK app files (`cdk.json`, entrypoints, etc.) | **task-output** (via `cdk init`) | All files generated by `cdk init app --language=<cdk_language>`. Guard: skip if `<placement_dir>/cdk.json` already exists. Becomes user-owned after first run. |
| `<placement_dir>/cdk.context.json` | **committed** (not gitignored) | Part of reproducible infra state; staged to git automatically after init. |
| `<placement_dir>/cdk.out/` | **gitignored** | Synth output — not committed. |

## Questions

| Key | Type | Default | Notes |
|---|---|---|---|
| `project_name` | str | `_external_data.base.project_name` | Read from `bailiff-mod-base` via `_external_data` alias; used in cdk init. |
| `cdk_language` | str (choices) | `typescript` | `typescript`, `python`, `go`, `java`, `csharp`. |
| `placement_dir` | str | `infrastructure` | Where `cdk init` writes. Use `.` for standalone IaC repos. |
| `include_cdk_nag` | bool | `false` | Splice `cdk-nag` import into the app entrypoint. |
| `include_synth_validate` | bool | `false` | Run `cdk synth` for offline empty-stack validation. |

## De-opinionation

- **No hardcoded account/region** — use `CDK_DEFAULT_ACCOUNT`/`CDK_DEFAULT_REGION` env vars
  or omit; never baked into the scaffold.
- **No default VPC, Lambda, or any opinionated stack** — `cdk init` produces an empty
  `CdkStack`; users add their own constructs.
- **NEVER `cdk bootstrap` or `cdk deploy`** — those are irreversible actions requiring
  cloud credentials; must be run explicitly by the user.

## Task chain

1. **Preflight** — verify `mise`, `node`, and `cdk` are on PATH; fail with install URLs.
2. **`cdk init app --language=<cdk_language>`** — init-only-guarded: skip if
   `<placement_dir>/cdk.json` already exists (idempotent reproduce).
3. **Commit `cdk.context.json`** — staged to git when present (no-op otherwise).
4. **`cdk synth`** (opt-in, `include_synth_validate=true`) — credential-free validation
   of an empty stack. Never runs on reproduce.

## Ordering & dependency

- `depends_on: [bailiff-mod-base]`, `phase: normal` (spec 014 single-edge model).
  bailiff enforces base is present and ordered before cdk (hard data-dependency via
  `_external_data` alias). NO edge to language modules: `cdk init` is self-contained in
  `placement_dir` (contract: "NO edge to language modules").
- `project_name` reads from `_external_data.base.project_name` (spec 014 / FR-004);
  base is a hard requirement enforced at bailiff preflight.

## Prerequisites (FR-007b)

This template runs trust-gated `_tasks`, so the source must be trusted. Preflight checks:

- **mise** — <https://mise.jdx.dev>
- **node** — `mise use node@lts`
- **cdk** — `npm install -g aws-cdk`

## Usage

Prefer bailiff (multi-layer, in dependency order):

```sh
uvx bailiff init --run-spec <run-spec with [bailiff-mod-base, bailiff-mod-cdk]>
```

Copier-only (standalone; renders with defaults):

```sh
copier copy --trust https://github.com/bailiff-io/bailiff-mod-cdk.git <destination>
```
