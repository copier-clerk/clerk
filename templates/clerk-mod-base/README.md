# clerk-mod-base

The collapsed clerk **base scaffold** — one copier template that ports the six
always-on [`project-setup`](https://github.com/srobroek/project-setup) base
modules (core-identity, dirs-scaffold, gitignore-generate, license-write,
agents-md, git-init) into a single reproducible layer (spec 009 Phase 0, Q1 /
FR-013). Every other `clerk-mod-*` layer (languages, quality, tooling)
`run_after` this module.

## What it produces

| Output | Lifecycle | Notes |
|---|---|---|
| Directory scaffold (`<dir>/.gitkeep`) | **managed** | 20 base dirs; +15 monorepo targets when `layout=monorepo`. Re-rendered byte-identically at reproduce. |
| `AGENTS.md` | **seed-once** (`_skip_if_exists`) | `single`/`monorepo` body chosen by `layout`; `## Architecture` span filled from the frozen `architecture_md` answer iff `write_architecture=true`. Scaffolded once, then project-owned. |
| `.gitignore` | **task-output** (gitnr) | Generated by the version-pinned `gitnr` task from the `gitignore_stack` answer. Process-deterministic, regenerated on reproduce. |
| `LICENSE` | **task-output** (`gh api`) | Fetched from the GitHub Licenses API; `[year]`/`[fullname]` filled from the frozen `today`/`org`. Outside the byte-identical set. |
| `.git/` + optional initial commit | **task-output** (git) | `git init`; then, iff `initial_commit=true`, `git add -A && git commit`. |
| `.copier-answers.yml` | **managed** | Records `_src_path` + `_commit` for faithful reproduce. |

## Prerequisites (FR-007b)

This template runs trust-gated `_tasks`, so the source must be trusted
(`clerk trust add …`) before it renders. A preflight task (ordered first) checks
these tools and fails with install guidance if any is missing:

- **git** — <https://git-scm.com/downloads>
- **gh** — <https://cli.github.com>; the LICENSE task needs it authenticated
  (`gh auth status`). No token is stored in copier answers — `gh` reads ambient
  credentials (Constitution VI / FR-002).
- **gitnr** — <https://github.com/reemus-dev/gitnr>; pinned to **0.3.0** in the
  `.gitignore` task (a version mismatch warns loudly).

## Usage

Prefer clerk (handles trust, ordering, answer threading, and `today` injection):

```sh
uv run scripts/clerk.py init --run-spec <run-spec.(json|yml)>
```

Copier-only (single layer):

```sh
copier copy --trust https://github.com/copier-clerk/clerk-mod-base.git <destination>
```

## Notes for the implementer (residual, spec 009 tasks.md)

- **git-commit scope**: project-setup's `git-init` declares soft `after`-edges over
  *every* module (including later-phase ones) so the commit runs last. In the
  collapsed Phase-0 base the commit is simply the last base `_task`, which is
  faithful for a `[base, python]` selection. If a later layer must be committed
  by base, revisit once that layer lands.
- **gitignore stack threading**: base owns the single `.gitignore` writer; language
  overlays contribute their token into the shared `gitignore_stack` answer (see
  `clerk-mod-python`). No second writer, no non-idempotent append.
