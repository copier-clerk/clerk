# Tasks: bailiff Single-Template Vertical Slice

**Input**: Design documents from `specs/001-bailiff-vertical-slice/`
**Prerequisites**: [plan.md](./plan.md) (required), [spec.md](./spec.md) (user stories),
[constitution](../../.specify/memory/constitution.md) v2.0.0

**Tests**: INCLUDED. Constitution VII makes per-step hardening (determinism,
error-surfacing, static-discovery-safety) part of this slice's definition-of-done,
so test tasks are first-class here, not optional.

**Organization**: grouped by user story (US1–US6 from spec.md) so each is
independently testable.

## Code-layout decision (reconciles plan.md)

The lean model = **skill + example template + minimal glue**. The glue reuses the
already-scaffolded `src/bailiff/` package (existing `bailiff` entry point, mypy/ruff/
pytest already configured) as a small CLI — **no pydantic, no deprecated-surface
adapter, no committed JSON schema**. Modules: `cli.py` (verb dispatch),
`discovery.py` (static `copier.yml`/file-tree/tag parse), `trust.py` (settings.yml
read/add/list), `runner.py` (thin wrappers over copier's public `run_copy`/
`run_recopy`). init/reproduce use copier's **public functions** (stable surface),
not the deprecated `Template`/`Worker`. The example template lives at
`examples/bailiff-template-example/`; the skill at `skills/bailiff/SKILL.md`.

- **[P]**: can run in parallel (different files, no incomplete-task dependency)
- **[Story]**: US1–US6 (user-story tasks only)
- Exact file paths included

## Dependencies & Execution Order

### Phase order

- **Setup (P1)** → **Foundational (P2)** blocks everything (need the template + fixtures).
- **US3 discover** first among stories — it is the agent's input and feeds the run-spec.
- **US4 trust** next — init needs trust to run the action-taking template.
- **US1 init** depends on US3 (discovery informs the doc) + US4 (trust) mechanically.
- **US2 reproduce** depends on US1 (needs a produced project).
- **US5 refusal** extends US3/US1's gate.
- **US6 check** extends US1's init path.
- **Polish** last (docs, skill, quickstart, smoke, full verify).

### The MVP is the P1 set (US3+US4+US1+US2)

Unlike a typical "US1 alone" MVP, this slice's minimal honest loop is the four P1
stories together — discover → trust → init → reproduce. US5/US6 (P2) harden it.

### Within a story

Tests are written first and must fail before implementation (TDD; Constitution VII).
`discovery.py`/`trust.py` before the `cli.py` verbs that call them; `runner` before
its CLI verb.

### Parallel opportunities

- Setup T002/T003 parallel; Foundational T005–T008 parallel.
- Within each story the `[P]` test tasks run together, then the (serial)
  implementation tasks in that story.
- Polish T031–T034 are all `[P]`.

## Implementation Strategy

1. Setup + Foundational → the template + fixtures + harness exist.
2. US3 discover → US4 trust → US1 init → US2 reproduce → **STOP & VALIDATE the full
   loop** (this is the MVP: a reproducible project, agent-free reproduce).
3. US5 + US6 harden (refusal + check).
4. Polish: docs fix, SKILL.md, quickstart, smoke test, full `test/types/lint`.
