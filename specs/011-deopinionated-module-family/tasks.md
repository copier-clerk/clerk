## Module Definition of Done (applies to EVERY module task)

Every module task below implicitly includes ALL of the following (from `specs/011-deopinionated-module-family/contracts/_cross-cutting.md`); do not consider a module task complete without them:

1. **Module shape**: `templates/bailiff-mod-<name>/copier.yml` with `_subdirectory: template`, `template/{{ _copier_conf.answers_file }}.jinja`, `README.md`, `CHANGELOG.md` (with the `- - -` cocogitto separator). New modules start from `just new-module name=bailiff-mod-<name>` (scaffold in `_meta/module-template/`); the reference shape is `templates/bailiff-mod-python/`.
2. **Registration parity**: `just check-modules` green — three-way parity `templates/` == `cog.toml [monorepo.packages]` == `catalog-sources.toml` (gate: `scripts/check_modules.py`, SC-007). Also register in `skills/bailiff/SKILL.md` where applicable.
3. **Loop test**: hermetic init + reproduce test under `tests/loop/` with ALL native/network tasks stubbed to offline marker writes via the existing `_copy_module_with_stub_tasks` fixture in `tests/conftest.py`. Assertion mapping (R5): **config-assert** MANAGED renders on init AND reproduce; **presence/structure-assert** TASK-OUTPUT (never assert regeneration); **`_skip_if_exists`-assert** SEED-ONCE (populated-tree re-run never clobbers edits).
4. **Determinism/trust**: NO `secret:` questions anywhere (tests/loop/test_secrets_policy.py must stay green); NO `jinja2_time` — `today` arrives as an injected answer; code/network steps are trust-gated `_tasks` with the mise preflight ordered FIRST (FR-009); tool versions pinned via mise.
5. **Init-only guards (FR-012a / M3)**: every preflight (`mise install`) and native-init `_task` is guarded (`test -f <manifest> || <tool> init …`, or a committed sentinel) so reproduce over a committed tree never re-shells and succeeds without toolchain or network.
6. **Frozen-union single-writer (M1)**: shared accreting files are never runtime-accumulated. Contributors only add tokens/blocks to agent-frozen union answers injected via `--data`: `gitignore_stack`/`mise_tools` → written only by bailiff-mod-base; `hook_manager`+`hook_blocks` → written only by bailiff-mod-precommit; `quality_languages` → written only by bailiff-mod-quality. Threaded axes use `default: "{{ <key> }}"` (FR-010).
7. **Native-command scaffolding (FR-007 / ADR-0007)**: initial manifests come from the tool's own init under mise-pinned versions (task-output, then seed-once); later deps via native `add` commands, never manifest edits; NEVER an irreversible action at scaffold (`cdk bootstrap/deploy`, `terraform apply`, `sam deploy`, unconsented public `gh repo create`).
8. **C-11 / FR-011 gate**: NO new `src/bailiff/` code and NO new `scripts/bailiff.py` verb — all behavior is pure template content under `templates/bailiff-mod-<name>/` (copier questions + rendered files + trust-gated tasks).

### Why the intra-slice ordering

- **T005 precommit second**: owns the `hook_manager`+`hook_blocks` frozen-union single-writer contract (M1) that python/ts/go/rust contribute blocks to and quality/justfile read — the contract must be frozen before its consumers are authored.
- **base → python serial**: both are MAJOR clean-break revisions of released modules; python threads base answers and adds precommit to run_after.
- **Slice B after A**: readme/stack-adr/ci sort lexicographically BEFORE language layers, so their sizing facts (ci_languages, ci_lang_facts, stack_pins, readme_body) must be agent-frozen `--data` derived from finished Slice A modules (FR-010).
- **Slice C last**: integrates with core + CI family (package-add uses the language modules' native add idioms; github-repo/IaC are overlays on the finished base).
- **Phase 7 strictly serial and gated**: mirror pre-creation must precede merge (the armed pipeline fans out to mirrors); tombstone must be sequenced with catalog regen (R6).
